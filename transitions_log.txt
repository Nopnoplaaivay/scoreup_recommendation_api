gửi body request 1 dãy các câu vừa gợi ý để train model (T nghĩ tối đa nên là 10 bài tập thôi)
Làm xong 10 câu sẽ lại chạy lại cái api lấy câu mới từ đầu

Ví dụ làm xong bài 1 gợi ý bài 2 thì transition là bài 1 -> 2
VD: [1->2] [2->3] ... [9->10]

Sau khi thằng học làm xong 1 chuỗi 10 câu mình vừa gợi ý đó. Gọi api này rồi truyền body request như sau 
để train model:

{
    "user_id": "669d16e11db84069209550bd",

            // State chứa exercise_id, difficulty, score, bookmarked, knowledge concept

            "state": [

                0, // điểm

            ],

        {
            // Đây là bài 2 -> bài 3

                "3b6fd0b3-414b-464e-83f1-404d78be26c9",

* Note: trong quá trình làm, thằng user có thể thoát ra không làm nữa, nếu nó làm được 1 transition,
nghĩa là làm xong bài 1, nó được gợi ý bài 2, nó làm xong bài 2 (có score rồi) mà nó thoát ra thì cứ gửi 
body request như trên cho t. Nếu m bắt được sự kiện đấy để làm gán giá trị cho thằng done thì ngon.
VD: nó làm hết 10 câu mình gợi ý => done = 1, gửi về cho T
    hoặc nó làm 1-2 transition, nghĩa là 2-3 bài liền mạch rồi thoát ra thì cũng gửi body request như
    trên cho t.
